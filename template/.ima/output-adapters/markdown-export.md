# Output Adapter: Markdown Export

## When to Use
Default output format. Used for rich reports, analysis documents, and research deliverables.

## Instructions for AI Agent

When formatting IMA output as Markdown:

### File Structure
Save all outputs to the configured output directory (default: `outputs/`):
```
outputs/
├── discovery/
│   ├── [niche]-shortlist-[date].md
│   └── [niche]-profiles-[date].md
├── audits/
│   ├── [influencer-name]-vetting-[date].md
│   └── [brand-name]-audit-[date].md
├── campaigns/
│   ├── [campaign-name]-proposal-[date].md
│   └── [campaign-name]-brief-[date].md
├── competitive/
│   └── [niche]-competitive-analysis-[date].md
├── outreach/
│   └── [campaign-name]-outreach-[date].md
└── performance/
    └── [campaign-name]-report-[date].md
```

### Formatting Rules
1. Use proper heading hierarchy (H1 for title, H2 for sections, H3 for subsections)
2. Use tables for structured data comparisons
3. Use checkboxes for actionable items
4. Use bold for key metrics and important findings
5. Use emoji indicators for ratings (✅ 🟡 🟠 🔴 ⭐)
6. Include a table of contents for reports longer than 3 sections
7. Use blockquotes for key insights or callouts
8. Use horizontal rules to separate major sections

### Report Template
```markdown
# [Report Title]

> **Generated by IMA Framework** | Date: [YYYY-MM-DD] | Niche: [niche]

## Executive Summary
[2-3 paragraph overview]

## [Section 1]
[Content with tables, data, analysis]

## [Section 2]
[Content]

## Recommendations
1. [Actionable recommendation]
2. [Actionable recommendation]

## Next Steps
- [ ] [Action item]
- [ ] [Action item]

---
*Generated by IMA Framework v1.0*
```

### File Naming Convention
`[type]-[subject]-[YYYY-MM-DD].md`

Examples:
- `discovery-sustainable-fashion-2026-03-10.md`
- `vetting-jane-smith-2026-03-10.md`
- `campaign-spring-launch-2026-03-10.md`
