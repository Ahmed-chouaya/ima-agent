# Obsidian MOC (Map of Content) Template

Use this template when creating `_index.md` files for Obsidian vault output.

## Client Root MOC

```markdown
---
type: moc
client: "[Client Name]"
created: "[date]"
tags: [moc, client, [niche]]
---

# [Client Name] — Research Hub

> Generated by IMA Framework on [date]

## 📊 Overview
- **Niche:** [niche]
- **Platform(s):** [platforms]
- **Influencers Found:** [count]
- **Vetted:** [count]
- **Outreach Drafted:** [count]

---

## 🔍 Discovery
<!-- List discovered influencer profiles with wikilinks -->
- [[discovery/@handle1]] — ⭐ [score] — [tier] — [followers] followers
- [[discovery/@handle2]] — ⭐ [score] — [tier] — [followers] followers

## ✅ Audits
<!-- List completed audit reports -->
- [[audits/@handle1-audit]] — Status: [Passed/Flagged]
- [[audits/@handle2-audit]] — Status: [Passed/Flagged]

## 📧 Outreach
<!-- List outreach drafts -->
- [[outreach/@handle1-outreach]] — Status: [Draft/Sent/Responded]

## 📈 Campaigns
<!-- List campaign plans -->
- [[campaigns/campaign-name]] — Status: [Planning/Active/Complete]

## 📊 Reports
<!-- List performance reports -->
- [[reports/report-name]] — Date: [date]
```

## Subdirectory MOC

For subdirectories (discovery/, audits/, etc.), create a simpler `_index.md`:

```markdown
---
type: moc
section: "[discovery|audits|outreach|campaigns|reports]"
client: "[Client Name]"
tags: [moc, [section]]
---

# [Section Name]

> [Count] items | Last updated: [date]

## Files
- [[file1]] — [brief description]
- [[file2]] — [brief description]
```

## Individual File Frontmatter

Every file in an Obsidian vault should have YAML frontmatter:

### Influencer Profile
```yaml
---
type: influencer-profile
name: "[Name]"
handle: "@[handle]"
platform: [platform]
profile_url: "[full URL]"
followers: [number]
engagement_rate: [number]
tier: [nano|micro|mid|macro]
score: [1-10]
tags: [[niche], [tier], [platform], vetted|unvetted]
created: [date]
status: [discovered|vetted|contacted|contracted]
---
```

### Audit Report
```yaml
---
type: audit-report
influencer: "[[discovery/@handle]]"
result: [passed|flagged|failed]
risk_level: [low|medium|high|critical]
tags: [audit, [platform], [result]]
created: [date]
---
```

### Outreach Draft
```yaml
---
type: outreach-draft
influencer: "[[discovery/@handle]]"
audit: "[[audits/@handle-audit]]"
method: [email|dm|management]
status: [draft|sent|followed-up|responded]
tags: [outreach, [method], [status]]
created: [date]
---
```
